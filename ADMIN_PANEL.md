# Admin Panel Documentation

## Overview

The admin panel is a complete management system for viewing, filtering, and managing all registered candidates. All data is stored in Firebase Firestore and synced in real-time.

## Features

### 1. Dashboard Statistics
- **Total Candidates**: Count of all registered candidates
- **Successful Payments**: Count of candidates with successful payment status
- **Placed Candidates**: Count of candidates with "placed" status
- **Total Revenue**: Sum of all successful payments (₹1000 per candidate)

### 2. Candidate Management
- **Real-time List**: All candidates displayed in a sortable table
- **Search**: Filter by name, email, or phone number
- **Status Filters**: Filter by payment status (all, pending, success, failed)
- **Detailed View**: Click any candidate to see full profile

### 3. Candidate Details
- Personal information (name, father's name, email, phone)
- Address information (current & permanent)
- Job position and registration date
- Payment details and transaction ID
- Uploaded documents (resume, PAN card)
- Status update capability

### 4. Status Management
Update candidate status through the admin panel:
- **Registered**: Initial status after payment
- **Confirmed**: After verification
- **Placed**: After successful job placement

## Accessing Admin Panel

### Login
1. Click the **"Admin"** button in the top navigation
2. Enter password: `admin123`
3. You'll be redirected to the admin dashboard

### Exit
Click the **"Exit Admin"** button in the bottom-right corner to return to the main site

## Data Structure

### Firestore Collection: `candidates`

```typescript
{
  id: string;                    // Auto-generated by Firestore
  name: string;                  // Full name
  fatherName: string;            // Father's name
  phone: string;                 // Phone number
  email: string;                 // Email address
  position: string;              // Desired job position
  currentAddress: string;        // Current address
  permanentAddress: string;      // Permanent address
  notes?: string;                // Additional notes
  resumeUrl?: string;            // S3 URL to resume
  panUrl?: string;               // S3 URL to PAN card
  paymentStatus: 'pending' | 'success' | 'failed';
  paymentId?: string;            // Razorpay payment ID
  paymentAmount: number;         // Amount in rupees (1000)
  status: 'registered' | 'confirmed' | 'placed';
  createdAt: string;             // ISO timestamp
}
```

## Real-time Updates

The admin panel uses Firebase's real-time listeners (`onSnapshot`). This means:
- New registrations appear instantly
- Status updates are reflected immediately
- No need to refresh the page
- Multiple admins can work simultaneously

## Security

### Password Protection
- Admin panel is protected with a simple password
- Default password: `admin123`
- **Change this in production!**

To change the password, edit `src/App.tsx`:
```typescript
if (password === 'admin123') {  // Change this
  setIsAdminMode(true);
}
```

### Firebase Security Rules
Recommended Firestore rules:

```
match /candidates/{document=**} {
  allow read: if request.auth != null;
  allow create: if request.auth != null;
  allow update: if request.auth != null && request.auth.uid == resource.data.adminId;
}
```

## Troubleshooting

### Admin Panel Not Loading
- Check Firebase connection
- Verify Firestore database is active
- Check browser console for errors

### Candidates Not Appearing
- Verify candidates collection exists in Firestore
- Check Firebase security rules allow read access
- Ensure real-time listener is active

### Status Update Not Working
- Verify Firebase write permissions
- Check candidate ID is correct
- Look for errors in browser console

### Search Not Working
- Ensure search term matches exactly
- Try searching by different field (name vs email)
- Clear search box and try again

## API Integration Points

- `collection(db, 'candidates')` - Access candidates collection

- `onSnapshot()` - Real-time updates
- `updateDoc()` - Update candidate status

### AWS S3

- Clickable in candidate detail view
